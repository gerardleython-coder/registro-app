# CONTRIBUTING

## Flujo de trabajo
- Crear ramas feature/ o fix/ por cambio.
- Commits descriptivos; abrir PR con título y descripción clara.

## Estándares
- Código formateado con Black; lint con Ruff; tipos validados con mypy.
- Pruebas con pytest; cobertura objetivo ≥ 80%.
- Documentación actualizada en cada cambio relevante.

## Revisión
- PRs deben incluir: qué se cambió, por qué, cómo se probó.
- Ejecutar linters y pruebas antes de solicitar revisión.
